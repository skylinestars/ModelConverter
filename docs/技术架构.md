# 三、基础类型约定

## ObjectID（统一 ID 体系）

```cpp
using ObjectID = uint64_t;
constexpr ObjectID INVALID_ID = ~ObjectID(0);
```

**设计原则：**

- 保留 uint64，为 BIM / 点云 / 大规模实例化场景留余量
- 所有跨对象引用（Node→Mesh、Node→Parent、Skin→Skeleton 等）统一用 ObjectID
- **不使用 shared_ptr / unique_ptr 做跨组件引用**，避免 DLL 边界、序列化、多线程遍历的复杂性

**关键约定：ObjectID 与 Vector Index 解耦**

```text
ObjectID 永不复用  —  生命周期内稳定，适合序列化 / undo / redo / 跨 Pass 引用
Vector Index 可变化  —  删除、合并、排序后 index 会变
```

**实现方式（代码阶段决定）：**

- 方案 A（简单）：`std::vector<Node> nodes` 按顺序 push_back，ObjectID 由全局计数器分配，每个对象内部 `id` 字段即其 ID。通过辅助函数 `Node* FindNode(ObjectID)` 做线性查找，或 Scene 维护 `unordered_map<ObjectID, size_t> nodeIndexById`。
- 方案 B（高性能）：`std::vector<Node> nodes` + `std::unordered_map<ObjectID, size_t>` O(1) 映射表，删除节点时只更新映射表而不触发元素移动。

**无论哪种方案，对外接口一律用 ObjectID 做引用，绝不暴露 vector index。** 后续做节点删除、撤销重做、节点合并、节点排序时，ID 与 Index 绑定的设计会彻底卡住。

## Result / Expected（错误机制）

```cpp
// 简化版：ok/value/error 三段式。将来可以无损替换为 std::expected（C++23）或 tl::expected。
template <class T>
struct Result
{
    bool                        ok       = false;
    T                           value    {};
    std::string                 error;                  // 致命错误（ok=false 时必填）
    std::vector<std::string>    warnings;               // 非致命问题（ok=true 时也可能存在）

    explicit operator bool() const { return ok; }
};

// 无返回值的场景
template <>
struct Result<void>
{
    bool                        ok = false;
    std::string                 error;
    std::vector<std::string>    warnings;

    explicit operator bool() const { return ok; }
};

using VoidResult = Result<void>;
```

所有 Importer / Exporter / Pass 操作统一返回 Result，而不是裸 bool。

**错误信息规范**：
- `error`：致命错误（操作失败）。格式：`[Location] reason (context: xxx)`
- `warnings`：非致命问题（如：某个顶点权重和 = 0.98，已由 Pass 补 1.0；某个 Material 引用了不存在的 Texture，已用默认材质替代）
- `warnings` 即使在 `ok=true` 时也可能非空——调用方应检查并记录

# 四、统一场景数据结构

## Scene（森林 + 平铺存储）

```cpp
struct Scene
{
    SceneMetadata              metadata;

    // SceneGraph（森林）
    std::vector<ObjectID>      rootNodes;    // 多个根节点
    std::vector<Node>          nodes;        // 所有节点平铺存储

    // Geometry（与 SceneGraph 分离，Node 只存 ObjectID 引用）
    std::vector<Mesh>          meshes;

    // Material / Texture
    std::vector<Material>      materials;
    std::vector<Texture>       textures;

    // Skeleton / Skin（三分离：Node ≠ Bone ≠ Skin）
    std::vector<Skeleton>      skeletons;
    std::vector<Skin>          skins;

    // Animation
    std::vector<AnimationClip> animations;

    // 实例化（USD PointInstancer / FBX Instance）
    std::vector<PointInstancer> pointInstancers;

    // 其他
    std::vector<Camera>        cameras;
    std::vector<Light>         lights;
};

// ---------- Camera / Light（骨架，阶段 2 扩展） ----------
enum class CameraType { Perspective, Orthographic };

struct Camera
{
    ObjectID    id;
    std::string name;
    CameraType  type       = CameraType::Perspective;

    // Perspective
    float       yfov       = 1.047f;   // 垂直 FOV（弧度）~60 度
    float       aspectRatio = 1.0f;
    float       znear      = 0.1f;
    float       zfar       = 100.0f;

    // Orthographic（type == Orthographic 时才用）
    float       xmag       = 1.0f;
    float       ymag       = 1.0f;
};

enum class LightType { Point, Directional, Spot, Ambient };

struct Light
{
    ObjectID    id;
    std::string name;
    LightType   type       = LightType::Point;
    Vec3        color      = Vec3(1, 1, 1);
    float       intensity  = 1.0f;

    // Point / Spot
    float       range      = 0.0f;     // 0 = 无限

    // Spot
    float       innerConeAngle = 0.0f;
    float       outerConeAngle = 0.5f;
};
```

**关键设计决策：**

1. **森林结构**：`rootNodes` + `nodes`，符合 GLTF/USD/FBX 多根语义，不人为合成虚拟 Root
2. **Geometry / SceneGraph 分离**：Node 只存 `meshIds`，Mesh 是大对象统一由 Scene 拥有。这是实例化（一个 Mesh 被 1000 个 Node 引用）的基础
3. **三分离**：Node / Skeleton / Skin 是三套独立数据，Bone ≠ Node
4. **平铺存储**：所有对象都是 `std::vector<T>`，index 与 ObjectID 一一对应（或维护 `unordered_map<ObjectID, index>`）

# 六、Node 系统

```cpp
enum class NodeType { Empty, Mesh, Bone, Camera, Light, Helper };

struct Node
{
    ObjectID              id;
    ObjectID              parent;           // INVALID_ID 表示根
    std::vector<ObjectID> children;

    std::vector<ObjectID> meshIds;          // 引用 Scene::meshes（支持一个 Node 挂多个 Mesh）

    // NodeType::Camera / Light 时引用对应对象
    ObjectID              cameraId = INVALID_ID;  // 引用 Scene::cameras
    ObjectID              lightId  = INVALID_ID;  // 引用 Scene::lights

    // 预留：实例化相关（USD PointInstancer / FBX Instance / GLTF EXT_mesh_gpu_instancing）
    // 注意：此处 instanceOfNodeId 表示"本 Node 是某个原型的实例"（即 Node 自身是叶子实例）
    // PointInstancer::prototypeNodeId 是"Instancer 引用的模板"，两者职责不同
    ObjectID              instanceOfNodeId  = INVALID_ID;  // 原型节点；INVALID_ID 表示非实例

    std::string           name;
    NodeType              type;
    Matrix4               localMatrix;      // 相对父节点的变换
};
```

**关键点：**

- `parent` / `children` 全部用 `ObjectID`，不用裸指针或智能指针
- `meshIds` 是**引用**，不是拥有。一个 Mesh 可被任意数量 Node 引用（实例化天然成立）
- 不存 `worldMatrix`，需要时 BFS 上滚计算（避免存储冗余）
- **`localMatrix` 与动画 TRS 的优先级约定**：若该 Node 存在 `NodeAnimation`（即 `AnimationClip::nodeChannels` 中有对应条目），则运行时以 TRS 通道为准，`localMatrix` 仅作静态初始变换使用（GLTF 规范禁止同时使用 matrix 和 TRS，Core 层遵循同样约定：有动画时 TRS 优先）。Importer 导入带动画的节点时，应将静态变换拆解为 TRS 填入 `NodeAnimation`，而不是同时保留 `localMatrix`。

### PointInstancer（USD / FBX 大规模实例化）

当 Node 的 `instanceOfNodeId != INVALID_ID` 时，该节点就是一个"被引用的原型节点"，实际实例通过 `PointInstancer` 记录。

```cpp
struct PointInstancer
{
    ObjectID                  id;
    std::string               name;
    ObjectID                  prototypeNodeId;     // 引用哪个原型 Node（其下子树被整体实例化）

    // 每个实例的变换（逐实例）
    std::vector<Vec3>         positions;           // 每个实例的平移
    std::vector<Quaternion>   orientations;        // 每个实例的旋转
    std::vector<Vec3>         scales;              // 每个实例的缩放（可选）

    // USD 支持的多原型：可选 protoIndices[i] 指定第 i 个实例选哪个 prototype
    // 第一版不支持：prototypeNodeId 唯一；多原型扩展为 std::vector<ObjectID> prototypeNodeIds
    // + std::vector<int32_t> protoIndices
};
```

**MVP 阶段**：不实现读写。仅保留数据结构，供 USD/FBX 插件阶段 3-4 对接。

# 七、Mesh 结构（核心设计）

## 设计原则：标准属性强类型 + CustomAttribute 扩展槽

- 第一阶段（FBX/GLTF/OBJ/USD/STL）：**强类型标准属性**（positions/normals/tangents/uvs/colors），访问零 overhead
- 第二阶段（点云 LAS/PLY）：用 `customAttributes[]` 装 Intensity/Classification/GpsTime
- 第三阶段（BIM IFC）：用 `customAttributes[]` 装 FireRating/LoadBearing
- 第四阶段（USD Primvars）：再考虑是否升级为 VertexAttribute 大一统系统

## Mesh

```cpp
enum class PrimitiveType { Triangles, TriangleStrip, TriangleFan, Lines, Points, Polygons };

struct MeshSection
{
    uint32_t  indexOffset;
    uint32_t  indexCount;
    ObjectID  materialId;     // 引用 Scene::materials
};

struct MorphTarget
{
    std::string              name;
    std::vector<Vec3>        positionDeltas;   // 与 positions 同长（Importer 负责把稀疏数据展开）
    std::vector<Vec3>        normalDeltas;
    std::vector<Vec3>        tangentDeltas;
};

// ---------- CustomAttribute（扩展槽） ----------
enum class AttrFormat
{
    F32, F32x2, F32x3, F32x4,
    U8,  U32,  I32
};

struct CustomAttribute
{
    std::string           name;          // "temperature", "intensity", "classification" ...
    AttrFormat            format;
    uint32_t              elementCount;  // 通常等于 positions.size()
    std::vector<uint8_t>  data;          // 原始字节，按 format 解析
};
// 配套访问器（header-only）：
//   template <class T> span<const T> As(const CustomAttribute&)

// ---------- Skin（可变权重，不固定4关节） ----------
// Core 层允许任意数量的关节影响。工业模型（CAD / 扫描模型）常见 8+ 权重。
// 目标格式限制时由 LimitBoneInfluencePass 压缩为 N 权重（例如 GLTF 导出时压到 4）。
struct VertexInfluence
{
    uint16_t joint;     // Skeleton::bones 的 vector index（非 ObjectID，刻意的例外）
    // 原因：骨骼影响数据量大（每顶点多条），用 ObjectID 查找开销过高；
    // Skeleton::bones 在生命周期内不会重排，index 稳定。
    // LimitBoneInfluencePass 操作后 index 仍对应同一 Skeleton。
    float    weight;
};

// skinInfluences[vertexIndex] = 该顶点的所有骨骼影响（joint+weight 对）
// 每个顶点的 influences 数量可以不同；一般需要按 weight 排序后归一化。

struct Mesh
{
    ObjectID                                 id;
    std::string                              name;
    PrimitiveType                            primitiveType;

    // ============== 标准属性（强类型） ==============
    std::vector<Vec3>                        positions;
    std::vector<Vec3>                        normals;
    std::vector<Vec4>                        tangents;

    std::vector<std::vector<Vec2>>           uvs;      // uvs[set][vtx]，不写死 uv0/uv1
    std::vector<std::vector<Color4>>         colors;   // colors[set][vtx]

    std::vector<uint32_t>                    indices;
    std::vector<MeshSection>                 sections; // 多材质分段

    // 蒙皮：外层 index = vertex index；内层 = 该顶点的所有骨骼影响
    std::vector<std::vector<VertexInfluence>> skinInfluences;

    std::vector<MorphTarget>                 morphTargets;

    BoundingBox                              bbox;

    // ============== 扩展属性（第二阶段以后用） ==============
    std::vector<CustomAttribute>             customAttributes;
    // 预留下一代扩展槽：
    // std::vector<VertexAttribute>           vertexAttributes;   // 未来 USD-style Primvars
};
```

**核心决策：**

| 项目 | 决策 | 理由 |
|---|---|---|
| 多材质分段 | MeshSection[] | GLTF primitives / USD GeomSubset / FBX LayerElementMaterial 都需要 |
| UV/Color | vector of vectors | GLTF 支持多套 UV/Color，写死 uv0/uv1 会丢数据 |
| primitiveType | 显式枚举 | OBJ/STL 只有三角形，但 FBX/USD/GLTF 支持 strip/lines/points/polygons |
| MorphTarget | Mesh 内预留 | BlendShape 在 FBX/GLTF/USD 中非常常见，后期补会大改 |
| 扩展属性 | CustomAttribute[] | 为 LAS/IFC/USD 等非标准属性留出口，不拖慢第一版 |
| 智能指针 | 不使用 | DLL 边界、序列化、多线程遍历更简单；平铺 vector + ObjectID |

# 八、Texture 系统（支持外置文件 + Embedded）

```cpp
enum class TextureWrap
{
    ClampToEdge,
    Repeat,
    MirroredRepeat
};

enum class TextureFilter
{
    Nearest,
    Linear,
    NearestMipmapNearest,
    NearestMipmapLinear,
    LinearMipmapNearest,
    LinearMipmapLinear
};

struct TextureSampler
{
    TextureFilter  minFilter = TextureFilter::LinearMipmapLinear;
    TextureFilter  magFilter = TextureFilter::Linear;
    TextureWrap    wrapS     = TextureWrap::Repeat;
    TextureWrap    wrapT     = TextureWrap::Repeat;
};

struct Texture
{
    ObjectID         id;
    std::string      name;

    // 二选一：uri（外置） 或 embeddedData（内嵌）
    std::filesystem::path       uri;            // 相对路径或绝对路径
    bool                        embedded = false;
    std::vector<uint8_t>        embeddedData;   // PNG/JPG/WebP 原始字节
    std::string                 mimeType;       // "image/png", "image/jpeg" ...

    TextureSampler   sampler;
};
```

**设计要点：**

- **GLTF Embedded**：数据直接写入 `embeddedData`，导出时 base64 编码进 `data:` URI
- **FBX Embedded**：FBX 文件内部的 texture blob 写入 `embeddedData`
- **USD Asset**：USD 的 `asset path` 写入 `uri`
- **外置 PNG/JPG**：只填 `uri`，导出时按格式要求输出路径
- Importer 决定是否展开 embedded（可用 ImportOptions 控制）
- Exporter 决定是否打包 embedded（可用 ExportOptions 控制）

# 九、Material 系统（统一 Workflow 模式）

不做 PBRMaterial + PhongMaterial 双类，统一为一个 Material + Workflow 枚举。

```cpp
struct TextureRef
{
    ObjectID textureId = INVALID_ID;  // 引用 Scene::textures；INVALID_ID 表示"未使用"，ValidatePass 跳过
    int      uvSet     = 0;           // 使用哪一套 UV
    float    scale     = 1.0f;        // 数值贴图强度
};

struct Material
{
    enum Workflow
    {
        MetallicRoughness,   // GLTF / USD 标准
        SpecularGlossiness,  // FBX 常见
        Phong,               // 传统 OBJ/FBX
        Unlit                // GLTF KHR_materials_unlit

    ObjectID    id;
    std::string name;
    Workflow    workflow;

    // MetallicRoughness
    // 注意：metallic=1.0, roughness=1.0 是 GLTF 2.0 规范默认值
    // 调试阶段：实际使用时建议在 Importer 层面改为 (metallic=0, roughness=0.5)，
    // 否则未设置材质的 Mesh 会渲染为"全金属高粗糙度"的深色外观
    Vec4   baseColor         = Vec4(1, 1, 1, 1);
    float  metallic          = 1.0f;
    float  roughness         = 1.0f;

    // SpecularGlossiness
    Vec3   specular          = Vec3(1, 1, 1);
    float  glossiness        = 1.0f;

    // Phong（Legacy）
    Vec3   diffuse           = Vec3(1, 1, 1);
    Vec3   ambient           = Vec3(0, 0, 0);
    float  shininess         = 32.0f;

    // 公共
    float  opacity           = 1.0f;
    float  alphaCutoff       = 0.0f;   // 0 = 不使用 alpha test
    bool   doubleSided       = false;
    Vec3   emissive          = Vec3(0, 0, 0);

    // 贴图引用
    TextureRef baseColorTexture;
    TextureRef metallicRoughnessTexture;
    TextureRef normalTexture;
    TextureRef emissiveTexture;
    TextureRef occlusionTexture;
    TextureRef specularTexture;
    // ... 按需扩展
};
```

Importer 负责把源格式字段填入对应字段；Exporter 按目标格式的能力读取并做必要降级（如 MR→SG 互转）。

# 十一、动画系统（TRS + Morph + Visibility）

```cpp

enum class AnimationInterpolation
{
    Step,        // GLTF: STEP, FBX: Constant, USD: Held
    Linear,      // GLTF: LINEAR, FBX: Linear, USD: Linear
    CubicSpline  // GLTF: CUBICSPLINE, FBX: Cubic
};

// ---- KeyFrame ----
template <typename T>
struct KeyFrame
{
    double time  = 0.0;
    T      value = {};
};

// ---- AnimationTrack ----
template <typename T>
struct AnimationTrack
{
    AnimationInterpolation   interpolation = AnimationInterpolation::Linear;
    std::vector<KeyFrame<T>> keys;

    bool Empty() const
    {
        return keys.empty();
    }

    bool Valid() const
    {
        return keys.size() > 0;
    }
};

// ----- 通道：节点 TRS 动画 -----
using TrackVec3  = AnimationTrack<Vec3>;
using TrackQuat  = AnimationTrack<Quaternion>;
using TrackFloat = AnimationTrack<float>;

// ---- NodeAnimation: TRS + Visibility ----
struct NodeAnimation
{
    ObjectID    nodeId = INVALID_ID;

    TrackVec3   translation;
    TrackQuat   rotation;
    TrackVec3   scale;

    // Visibility: 0.0 = hidden, 1.0 = visible (USD / FBX)
    TrackFloat  visibility;
};

// ----- 通道：Morph / BlendShape 动画 -----
// 一个 MorphAnimation 控制 Mesh 上一个特定 MorphTarget 的权重
struct MorphAnimation
{
    ObjectID    meshId     = INVALID_ID;  // 引用 Scene::meshes
    // 注意：morphIndex 是 vector index 而非 ObjectID，这是刻意的例外。
    // 原因：MorphTarget 是 Mesh 的内嵌数组，不是 Scene 顶层对象，没有独立 ObjectID。
    // 实践中 MorphTarget 极少被删除或重排，常规操作下 index 稳定。
    uint32_t    morphIndex = 0;  // Mesh::morphTargets[morphIndex]
    TrackFloat  weights;
};

// ----- 动画片段 -----
struct AnimationClip
{
    ObjectID                    id;
    std::string                 name;

    //有些动画：100ms~200ms，所以不能用duration
    double startTime = 0.0;
    double endTime = 0.0;

    std::vector<NodeAnimation>  nodeChannels;
    std::vector<MorphAnimation> morphChannels;

    double Duration() const
    {
        return endTime - startTime;
    }
};
```

**关键说明：**

- `MorphAnimation` 与 `MorphTarget` 通过 `meshId + morphIndex` 绑定。这是 **FBX/GLTF/USD 表情动画（眨眼、张嘴、微笑）** 的核心数据结构。
- 之前设计只含 TRS，会导致表情动画数据直接丢失。
- Visibility 通道预留用于 USD/FBX 场景图节点显隐动画。
- Morph 动画常见于面部表情、布料/软体的 BlendShape 驱动。

# 十四、FormatRegistry + PluginManager

## Registry

```cpp
using ImporterCreator = std::function<std::unique_ptr<IImporter>()>;
using ExporterCreator = std::function<std::unique_ptr<IExporter>()>;

class FormatRegistry
{
public:
    static FormatRegistry& Instance();

    void RegisterImporter(const std::string& ext, ImporterCreator c);
    void RegisterExporter(const std::string& ext, ExporterCreator c);

    std::unique_ptr<IImporter> GetImporter(const std::string& ext) const;
    std::unique_ptr<IExporter> GetExporter(const std::string& ext) const;
};
```

## Plugin 统一接口（DLL 导出）

**不使用 `REGISTER_IMPORTER` 宏自动注册**（DLL 场景下静态初始化顺序、链接器裁剪都是坑）。工业软件统一做法是显式调用：

```cpp
// 每个 plugin 的 CMakeLists.txt 把这些导出为 C 符号
extern "C" {
    MC_PLUGIN_EXPORT const PluginInfo* GetPluginInfo();   // 返回 plugin 元信息（含 interfaceVersion 校验）
    MC_PLUGIN_EXPORT bool              RegisterPlugin(FormatRegistry& registry);
    MC_PLUGIN_EXPORT void              UnregisterPlugin(FormatRegistry& registry);
}
```

## PluginManager 加载流程

```cpp
class PluginManager
{
public:
    // 1. 扫描 plugins/ 目录下的 .dll / .so / .dylib
    // 2. LoadLibrary / dlopen
    // 3. GetProcAddress / dlsym("GetPluginInfo")，检查 interfaceVersion 是否匹配
    // 4. GetProcAddress("RegisterPlugin")，调 RegisterPlugin(registry) 完成注册
    VoidResult LoadAll(const std::filesystem::path& pluginDir);

    // 退出时：对每个 plugin 调 UnregisterPlugin，再 FreeLibrary
    void UnloadAll();
};
```

### 线程安全约定

`FormatRegistry::Instance()` / `Register...` / `LoadAll` 约定**只在主线程初始化阶段调用**，不由多线程并发调用。因此：
- `FormatRegistry` 内部不使用 `std::mutex`，避免不必要的锁开销
- 多线程场景下，所有注册行为在 `main()` 启动初期完成，后续查询（`GetImporter`/`GetExporter`）只读，无数据竞争

### 版本兼容性策略

```cpp
// Plugin 接口版本号（格式变更时递增）
constexpr uint32_t PLUGIN_INTERFACE_VERSION = 1;

struct PluginInfo
{
    std::string name;
    std::string version;       // "1.0.0"
    uint32_t    interfaceVersion;  // 必须 == PLUGIN_INTERFACE_VERSION，否则拒绝加载
    std::string description;
};
```

- `ObjectID` 全局计数器：每个 `Scene` 独立计数（从 1 开始），不需要持久化跨 Session
- `PLUGIN_INTERFACE_VERSION` 在 Core 打破 ABI/API 时递增；Plugin 不匹配时 `LoadAll` 返回 warning + 跳过该 plugin
- 第一版不做向前兼容：Core/Plugin 版本必须完全匹配

# 十五、Pipeline 处理层

```text
Importer → Scene → [Pass1 → Pass2 → Pass3 → ...] → Scene → Exporter
```

## IPass 接口

```cpp
class IPass
{
public:
    virtual ~IPass() = default;
    virtual VoidResult Execute(Scene& scene) = 0;
    virtual std::string Name() const = 0;
};
```

## 推荐的基础 Pass 清单（按执行顺序）

| Pass | 作用 | 阶段 |
|---|---|---|
| **ValidatePass** | **必须第一个执行**。检查：重复 ObjectID、父子循环引用、非法 Mesh/Material/Skeleton 引用、权重和 ≠ 1、索引越界、空 Mesh。拦截 Importer 脏数据 | 必需 |
| **TriangulatePass** | 将 n-gon / strip 统一转三角形（GLTF/STL 只认三角形） | 必需 |
| **UnitConvertPass** | cm→m、mm→m 等，读取 SceneMetadata.unit | 必需 |
| **AxisConvertPass** | Y-Up ↔ Z-Up | 必需 |
| **HandednessFlipPass** | Left-Handed ↔ Right-Handed | 必需 |
| **MergeDuplicateVerticesPass** | 顶点去重（OBJ 导入尤其有用） | 推荐 |
| **IndexGenerationPass** | 将未索引顶点流 → 有索引 | 推荐 |
| **LimitBoneInfluencePass** | 将每顶点骨骼影响数压到 N（默认 4），删除权重最小的影响后重新归一化。配合可变权重 Skin 设计，供 GLTF 等限制型格式导出 | 推荐 |
| **NormalGeneratePass** | 从 geometry 重建法线 | 可选 |
| **MikkTSpacePass** | Tangent 生成（PBR 贴图需要） | 可选 |
| **MaterialConvertPass** | MR↔SG↔Phong 工作流互转 | 可选 |
| **MeshMergePass** | 合并 Mesh 以减少 draw call | 可选 |
| **InstanceDetectPass** | 识别重复 Mesh → 转实例化 | 可选 |
| **AnimationOptimizePass** | 关键帧稀疏化 / 压缩（含 Morph 动画通道） | 可选 |

**ValidatePass 设计要点：**

```text
输入：Importer 产生的 Scene
检查：
  1. 所有 ObjectID 唯一，无 INVALID_ID 正向引用
  2. 父子图无循环引用（从 rootNodes 出发做 DFS 标记环）
  3. Node::meshIds[i] 均能在 Scene::meshes 中通过 ObjectID 找到对应项
  4. MeshSection::materialId 均能在 Scene::materials 中通过 ObjectID 找到对应项
  5. Skin::skeletonId 均能在 Scene::skeletons 中通过 ObjectID 找到对应项
  6. 每个顶点的 skin weight 之和 = 1.0 ± epsilon
  7. indices 数组最大值 < positions.size()
  8. UV/Color 通道长度 == positions.size()
  9. MorphTarget positionDeltas.size() == positions.size()（Importer 负责把稀疏数据展开成 dense）
  10. 所有 TextureRef::textureId 若 != INVALID_ID，则需在 Scene::textures 中通过 ObjectID 找到对应项
  11. PointInstancer::prototypeNodeId 若 != INVALID_ID，则需在 Scene::nodes 中通过 ObjectID 找到对应项
  12. NodeAnimation::nodeId 均需在 Scene::nodes 中通过 ObjectID 找到对应项
  13. MorphAnimation::meshId 均需在 Scene::meshes 中通过 ObjectID 找到对应项，且 morphIndex < meshes[meshId].morphTargets.size()
  14. Node::cameraId 若 != INVALID_ID，需在 Scene::cameras 中通过 ObjectID 找到对应项
  15. Node::lightId 若 != INVALID_ID，需在 Scene::lights 中通过 ObjectID 找到对应项
  16. Skin::meshId 若 != INVALID_ID，需在 Scene::meshes 中通过 ObjectID 找到对应项
输出：VoidResult（含警告 + 致命错误列表）
```

**原则：** ValidatePass 只发现问题，不自动修复。自动修复由后续专门 Pass（如 `WeightNormalizePass`）处理。
这样 Importer 只负责"读"，脏数据在 Pass 层面统一处理，避免每个 Importer 重复写检查逻辑。

# 二十一、架构红线（任何阶段都不允许破坏）

1. **Core 绝不依赖 Assimp / FBX SDK / OpenUSD**。第三方库只能出现在 plugin 目录内。
2. **Node 绝不内嵌 Mesh**。Geometry 与 SceneGraph 分离是实例化的基础，一旦破坏就回不去了。
3. **Bone ≠ Node**。Skeleton/Skin/Node 三分离，一旦 Bone 混入 Node 体系，GLTF/USD 的 skin 模型就对不上。
4. **Plugin 注册是显式调用 RegisterPlugin(registry)**，不靠全局对象静态初始化。DLL 下自动注册是永无止境的坑。
5. **跨对象引用只用 ObjectID**，不靠 shared_ptr。DLL 边界、序列化、undo/redo 都会感谢这个决定。
6. **ObjectID 永不复用，永不等于 vector index**。支持删除/合并/排序的唯一途径。
7. **Importer 只负责"读"，不做数据修正**。脏数据拦截交给 ValidatePass，修复交给后续专门 Pass。
8. **Skin 不固定 4 权重**。Core 层允许任意数量，格式限制由 LimitBoneInfluencePass 处理。

# 二十二、总结

按照这个架构，未来扩展 IFC、STEP、IGES、SKP、3DM、PLY、LAS、点云、3DGS 时：

- **Core 数据结构基本不需要大改**
- **只需新增 plugins/<format>/（Importer + Exporter）**
- **新增或复用 pipeline/ 中的 Pass**

可以支撑一个长期演进的工业级三维格式转换平台。

```text
Core + Plugin + Pipeline = N + N 而非 N × N
```
